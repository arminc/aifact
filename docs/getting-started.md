# Getting Started

## Quick start

1. Run `init` to scaffold the local workflow files.
2. Run `refresh-project-config` to align `work/project-config.md` with the repository.
3. Add or refine `AGENTS.md` so agent behavior and repo expectations are explicit.
4. Start new delivery work with `brainstorm`.
5. Record actionable work with `record-story`.
6. Run phases manually or use an orchestrator.

If you only remember one rule, remember this one: start with `brainstorm`, and treat the artifact on disk as the source of truth.
